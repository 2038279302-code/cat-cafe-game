## 一、v1.0 核心要点总结

### 1.1 架构概览

```

├── DOM       —— 静态界面骨架（开始界面 / 暂停层 / 游戏UI）
└── <script>
```

### 1.2 游戏状态对象 G

| 字段 | 类型 | 说明 |
|---|---|---|
| coins | number | 金币数 |
| lv / exp / expMax | number | 等级 / 经验 / 升级阈值（×1.5递增） |
| hap | number (0~100) | 猫咪心情，影响收益系数 |
| eng | number (0~100) | 服务质量（当前未消耗，预留） |
| totalCus / done | number | 今日顾客数 / 完成订单数 |
| combo / comboTmr | number / timer | 当前连击数 / 6秒重置计时器 |
| sel | string | 当前选中菜品 key |
| custs[] | Customer[] | 在场顾客数组 |
| making / makeT / makeTgt / makingItem | - | 制作状态锁，防止并发接单 |
| running / paused | bool | 游戏启动 / 暂停标志 |

### 1.4 收益计算公式

```
最终收益 = round( 基础收益 × 速度系数 × 连击系数 × 心情系数 )

速度系数：waitPct > 60% → 1.3x；> 30% → 1.0x；≤ 30% → 0.7x
连击系数：combo ≥ 10 → 2.0x；≥ 5 → 1.5x；≥ 3 → 1.2x；其余 → 1.0x
心情系数：0.9 + hap/500（范围约 0.9 ~ 1.1）
```

### 1.6 已修复的历史 Bug

| Bug | 现象 | 根因 | 修复方案 |
|---|---|---|---|
| 双 rAF 循环 | 顾客/猫咪突然全部消失 | initLoop + loop 并行，并发 filter 替换数组 | 合并为唯一 loop，running=false 时只绘制静态内容 |
| 遍历中修改数组 | 顾客超时时其他顾客闪消 | forEach 中直接赋值 G.custs=filter(...) | removeIds Set + 遍历后统一过滤 |
| showMsg 销毁 DOM | 消息条失效，后续消息不显示 | innerHTML='' 销毁了 #msgbox 元素 | 直接修改 textContent，保留节点 |
| 制作品项错误 | 制作途中换选品导致进度/收益计算乱 | drawMakeBar/completeOrder 用 G.sel 而非订单 | 接单时锁定 G.makingItem = ITEMS[c.order] |
| 连击加成计算错位 | 第3单拿不到x1.2加成 | 先判断 cb 再 combo++ | 先 G.combo++ 再计算 cb |

## 二、v2.0 迭代记录（已完成 · 2026-06）

> v2.0 围绕"游戏闭环感 + 留存动力 + 策略趣味"三个方向完成了首批高优先级落地。

### 2.1 ① 存档系统（localStorage）

**目标**：给玩家"积累感"，跨局记录最好成绩，开始界面展示历史最佳激励重玩。

**实现要点**：
- 存储 key：`catcafe_best_v1`，结构 `{coins, maxCombo, lv, grade}`
- 只保留各维度历史最高值（合并写入，不覆盖）
- 评级采用字典序比较 `gradeOrder={S:4, A:3, B:2, C:1}`
- 触发时机：`closeShop()` 打烊时写入；`initBestDisplay()` 页面加载和回到首页时读取展示
- 首次游戏无存档时历史最佳卡片隐藏（`display:none`），有记录后自动显示

**新增 G 字段**：
```js
G.maxComboThisRun  // 本局最高连击，用于存档比较
G.missedCus        // 本局超时漏单数，用于结算评级
```

---

### 2.2 ② 今日营业结算界面

**目标**：给游戏一个明确的"收尾仪式感"，补完从开始→游戏→结束的完整循环。

**触发方式**：控制栏新增「🌙 打烊结算」按钮（紫色调，与暂停/重开视觉区分）。

**评级算法（满分100分）**：

| 维度 | 权重 | 说明 |
|---|---|---|
| 完成率（done/total） | ×40 | 最重要，惩罚漏单 |
| 最高连击 | 30分封顶 | ≥10全分，≥5共20，≥3共12，否则5 |
| 当前等级 | lv×4，≤20 | 奖励坚持游戏 |
| 金币 | 最多10分 | ≥500全分 |

评级区间：S≥88 / A≥70 / B≥50 / C<50

**弹窗内容**：
- 今日营业额 / 服务顾客 / 完成订单 / 最高连击（四格统计）
- 打破历史记录时出现 `NEW` 闪烁徽章
- 个性化亮点文案（全勤无漏单 / 达成连击 / 摸猫次数 / 解锁新等级等）
- 「再来一局」直接重置；「回到首页」回开始界面并刷新历史最佳

---

### 2.3 ③ 顾客多样化（VIP + 急性子）

**目标**：增加场上的信息密度和瞬时决策，让每一桌都不一样。

**类型分布**：

| 类型 | 概率 | 等待时长 | 视觉标记 | 收益规则 |
|---|---|---|---|---|
| 普通 | 60% | 14~23s | 无 | 正常公式 |
| VIP 👑 | 20% | 耐心×1.5（21~34s） | 金色皇冠 + 金光圈 + 金色等待条 | +基础收益×50% 小费 |
| 急性子 ⚡ | 20% | 仅 9~11s | 橙色闪电 + 脉冲光圈 + 橙红等待条 | 成功完成×2 倍金币 |

**收益公式更新**：
```js
// 急性子
earned = earned * 2

// VIP
const tip = Math.round(it.p * 0.5)
earned += tip
```

**策略意图**：
- VIP 对时间要求宽松，适合在急单堆叠时优先搞定急性子再回来服务
- 急性子时间窗口极短，需要玩家优先处理，否则失去高额奖励
- 两种类型同时在场时，玩家必须做顺序取舍，制造真实的压力感

## 三、v3.0 中期玩法深度方案

> 核心命题：**让玩家从"反应型操作"进化到"策略型经营"**。
> 以下三个方向相互咬合——多制作台让金币消耗有了出口，金币升级反哺制作效率，猫咪技能给高压节奏提供喘息空间。

---

### 3.1 多单并行制作台

**痛点**：making 锁是当前体验的最大瓶颈。3 个顾客同时等待，玩家却只能做 1 单，剩余 2 桌只能干看着，缺乏调度感。

#### 核心设计

引入 `STATIONS[]` 数组替代单一的 making 锁，每个制作台是独立的状态机：

```js
// 制作台结构
{
  id: 0,           // 台编号
  busy: false,     // 是否占用
  makeT: 0,        // 剩余制作时间
  makeTgt: null,   // 绑定的顾客对象
  makingItem: null // 锁定的菜品
}
```

**解锁节奏**（与金币系统联动）：

| 制作台 | 解锁条件 | 说明 |
|---|---|---|
| 台①（默认） | Lv.1 | 初始已有 |
| 台②（中台） | 花 200 金币购买 | 核心跃升时机，约 Lv.3 前后 |
| 台③（快速台） | 花 500 金币购买 | 满配状态，Lv.6+ |

**UI 变化**：
- Canvas 底部制作区从 1 条进度条变为最多 3 条并排（标注台号和菜品 emoji）
- 有空闲台时，顾客气泡点击立即分配到最近的空台
- 所有台都忙时，点击顾客提示"制作台已满，请稍候"

**调度策略层**（给玩家的思考空间）：
- 急性子顾客到来时，玩家需要判断是否打断当前制作（目前不可打断，顺序固定）
- VIP 耐心充裕，可以放在"慢台"制作高价菜；急性子分配到"快台"做低价快单

#### 状态机变化（主循环改造）

```js
// loop() 中
STATIONS.forEach(s => {
  if (!s.busy) return;
  s.makeT -= dt;
  if (s.makeT <= 0) {
    s.busy = false;
    const tgt = s.makeTgt;
    s.makeTgt = null; s.makingItem = null;
    completeOrder(tgt, s.id);
  }
});
```

### 3.2 金币用途系统（咖啡机升级）

**痛点**：金币只是"分数"，赚到后没有消费欲望，缺乏目标驱动感。

#### 升级商店设计

触发方式：游戏 UI 新增「🏪 升级商店」按钮，点击弹出购买面板（不打断游戏，类似暂停覆盖层）。

**商品表（3条核心升级线）**：

**线一：制作台扩展**（直接影响并行能力）

| 等级 | 价格 | 效果 |
|---|---|---|
| 台② 解锁 | 200 金 | 可同时制作 2 单 |
| 台③ 解锁 | 500 金 | 可同时制作 3 单 |

**线二：咖啡机提速**（缩短制作时间，影响翻台率）

| 等级 | 价格 | 效果 |
|---|---|---|
| 基础机（默认） | - | 正常制作时间 |
| 进阶机 | 150 金 | 所有菜品制作 -15% |
| 专业机 | 350 金 | 所有菜品制作 -30% |
| 大师机 | 700 金 | 所有菜品制作 -45% |

**线三：等候区扩容**（增加同时在场顾客上限，目前最多3人）

| 等级 | 价格 | 效果 |
|---|---|---|
| 标准区（默认） | - | 最多 3 位顾客 |
| 扩容区 | 300 金 | 最多 4 位顾客（增加一个座位） |

**G 对象新增字段**：
```js
G.shopLevel = {
  stations: 1,    // 当前制作台数量（1/2/3）
  machineSpd: 1.0, // 制作时间系数（越小越快）
  maxSeats: 3     // 顾客上限
}
```

**经济平衡设计**：
- 台② 200金：玩家约完成 15~20 单后能买到，体验首次升级的成就感
- 提速 150金：买完立刻感受到节奏变化，正反馈强烈
- 等候区 300金：买完立刻来第 4 位顾客，刺激感强但压力也翻倍（高风险高回报）

### 3.3 猫咪技能成长线

**痛点**：摸猫只是个小彩蛋（+8心情），没有成长感，玩家很快就不再主动触发。

#### 亲密度系统

每次摸猫 +1 亲密度（`G.intimacy`），亲密度累积解锁被动技能，技能永久生效（本局内有效）。

**三条技能线**：

**技能 1 · 招呼天赋**（`G.intimacy >= 5`）
> 猫咪学会主动打招呼，顾客来得更快一点

- 效果：顾客生成间隔 `cusInt` 额外 -10%（在等级加成基础上叠加）
- 视觉：解锁后猫咪头顶出现小铃铛 🔔，每次新顾客到来时猫咪"摇铃"微动画

**技能 2 · 灵爪加速**（`G.intimacy >= 15`）
> 猫咪掌握了咖啡机的使用技巧，制作提速

- 效果：所有制作时间 -10%（叠加咖啡机升级的提速效果）
- 视觉：制作进度条颜色变为粉金渐变，进度更快的视觉感知
- 设计意图：与咖啡机升级形成叠加，高端玩家可以叠到 -40%~-55% 的极限状态

**技能 3 · 猫咪帮厨**（`G.intimacy >= 30`）
> 猫咪偶尔出手帮忙，自动完成一个简单订单

- 触发：每完成 8 单后，有 25% 概率触发（`G.catHelpCd` 计数器）
- 效果：随机自动完成当前等待中最早来的顾客订单（固定给基础收益，不算连击，不给经验）
- 视觉：触发时猫咪跑向目标顾客的小动画（canvas 中 catX 临时移位），顾客头顶出现 ✨ 气泡，消息条显示"猫咪帮忙啦！🐾 白得一单！"

**摸猫新反馈**：
- 未解锁时：显示距离下一技能还差多少亲密度（"再摸 X 次解锁新技能！"）
- 解锁时：触发成就弹窗 + 技能图标浮现动画

**G 对象新增字段**：
```js
G.intimacy = 0       // 累计摸猫亲密度
G.catSkills = {
  bell: false,       // 技能1：招呼天赋
  fastPaw: false,    // 技能2：灵爪加速
  autoHelp: false    // 技能3：猫咪帮厨
}
G.catHelpCd = 0      // 帮厨冷却计数
```
